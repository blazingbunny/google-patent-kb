## Key context

- **Target:** Search engine patents (G06F16 class) + ML cross-domain (G06N, G06F40)
- **Build:** vast.ai RTX 4090, ~$5 total, one-shot pipeline
- **Serve:** argus (Xeon E-2388G, 125 GB RAM, AVX-512, no GPU), CPU embedding at query time
- **Data source:** Google Patents Public Datasets on BigQuery (free tier, 1 TB/month)
- **Index:** turbovec IdMapIndex, 4-bit quantization, 1024-dim (bge-large-en-v1.5)
- **Metadata:** SQLite with patents + chunks tables
- **Not started yet:** No BigQuery runs, no vast.ai instances, no actual index built

4. Checkpoint resumption speed for large jobs
5. Whether to benchmark bge-large vs voyage-law-2 before committing

Additionally, review the spec for missing sections, incorrect assumptions,
performance targets that don't match real-world constraints, and any
architectural risk.
